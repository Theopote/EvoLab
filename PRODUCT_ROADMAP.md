# EvoLab 产品路线图

> **核心定位**: 方案阶段真工具 - 从生成、推敲、比较到表达的完整链条

---

## 📋 产品定位重申

EvoLab **不是**"把建筑模型做得越来越细"  
EvoLab **是**"把方案阶段从生成、推敲、比较到表达这条链做得越来越顺"

### 核心逻辑
1. **优先做"建筑师会反复打开"的功能** - 快速进项目、快速出方案、快速比选、快速汇报
2. **优先做"表达和推敲"** - 而不是"构件和台账"
3. **AI重点是"加速方案推敲"** - 而不是"替你画完"

---

## 🎯 P0: 必须最先做的能力

这些能力决定 EvoLab 能不能成为"方案阶段真工具"

### P0-1 | 导入与描摹中心（Import / Trace）

**为什么是P0**  
真实工作不是从空白outline开始，而是从旧图、PDF、手绘、参考案例开始。

**技术基础**  
✅ `dxf-parser` 已安装  
✅ `pdf-parse` 已安装  
✅ `pdf-page-info` API已存在  

**P0范围**
- [ ] 图片/扫描图导入
- [ ] 透视校正与比例标定
- [ ] 半自动描摹
- [ ] PDF导入与页面选择
- [ ] DXF基础轮廓/线段解析
- [ ] 导入结果转为可编辑平面

**不必做**
- ❌ 完整CAD图层语义重建
- ❌ 高精度门窗识别
- ❌ 施工图级自动识别

**成功标准**
- 导入成为产品第一入口之一
- 用户能把"已有资料"接入EvoLab开始推敲

---

### P0-2 | 受约束的平面生成（Plan Generation with Constraints）

**为什么是P0**  
必须从"文本生成"升级到"受program、zoning、floors、brief约束的生成"

**技术基础**  
✅ `PlanResultGrid`已传入program/zoning/floors/brief  
✅ `generatePlanPrompt`已有约束逻辑  
✅ `ProjectDomain`结构完整  

**P0范围**
- [x] 根据brief + program + zoning生成2~4个方案
- [ ] 同一层生成多个布局版本
- [ ] 自动生成每个版本的"设计逻辑摘要"
- [ ] 用户可锁定某些条件后再生成
- [ ] 支持"保留公共区位置，重生其余部分"

**不必做**
- ❌ 复杂非正交自由形态
- ❌ 深入到施工构造逻辑

**成功标准**
- 方案不是"随机AI输出"，而是可解释、可比较
- 平面生成后自然进入Compare和Presentation

---

### P0-3 | 方案比选工作台（Compare Workspace）

**为什么是P0**  
方案阶段的真实价值在"能快速比较，知道各自适合什么场景"

**技术基础**  
✅ `PlanResultGrid`已展示多维度指标  
✅ `SchemeCompareGrid`组件存在  
✅ `comparison hints`逻辑已有  

**P0范围**
- [ ] A/B/C多方案同屏
- [ ] 同层不同布局对比
- [ ] 差异高亮
- [ ] 自动摘要: 哪个更紧凑/采光好/适合汇报
- [ ] 一键选择主方案
- [ ] 一键进入Presentation生成"方案对比页"

**成功标准**
- Compare成为独立主模块
- 用户可在EvoLab内完成真正的方案决策讨论

---

### P0-4 | Copilot局部推敲工作流（Proposal / Patch / Accept）

**为什么是P0**  
最常见工作是"核心筒改一下"、"走廊收紧"，而不是"从零来一次"

**技术基础**  
✅ `ChangeSet`、`StoredCopilotProposal`已存在  
✅ `acceptedOperationIds`、`lockedElementIds`架构完整  
⚠️ `modifyPlanPrompt`仍返回完整PlanVersion，需优化  

**P0范围**
- [ ] 选区级AI修改
- [ ] 锁定对象不改
- [ ] Proposal差异预览
- [ ] Accept / Reject单项操作
- [ ] 自动说明"为什么建议这样改"

**技术改进**
- [ ] 修改`modifyPlanPrompt`返回增量操作而非完整版本
- [ ] 增强`PlanChangeProposal`结构

**成功标准**
- Copilot从"重生成器"升级为"方案助手"
- 用户敢在真实方案里使用AI修改

---

### P0-5 | Presentation / 汇报表达中心

**为什么是P0**  
EvoLab最重要的能力之一就是汇报文本制作，和平面生成同级重要

**技术基础**  
✅ `pptxgenjs` 已安装  
⚠️ `render-panel.tsx`停留在"生成brief"层级  

**P0范围**
- [ ] 自动生成设计概念说明
- [ ] 自动生成功能分区说明
- [ ] 自动生成流线说明
- [ ] 自动生成多方案对比文字
- [ ] 自动汇总体量/立面/结构/MEP概念说明
- [ ] 一键导出PDF/PPT初稿

**不必做**
- ❌ 花哨版式引擎
- ❌ 完整品牌模板系统

**成功标准**
- 建筑师能用EvoLab直接生成"拿去讲"的材料
- Presentation成为正式主模块，替代现有Render角色

---

## 🚀 P1: 非常重要，但在P0后推进

### P1-1 | 体量与楼层切分（Massing to Floors）

**价值**: 从体量自动切层，适合方案阶段差异化能力

**技术基础**  
✅ `Level`、`standardFloorGroups`、`verticalElements`已埋钩子

**P1范围**
- [ ] 从体量自动切层
- [ ] 标准层识别
- [ ] 裙房/塔楼/退台层分组
- [ ] 楼层轮廓批量套用平面逻辑
- [ ] 平面与体量双向联动

**为什么不是P0**: 没有Import/Plan/Compare/Presentation时，体量切层难构成完整闭环

---

### P1-2 | 立面与开窗概念优化（Facade Concept）

**价值**: 建立项目气质、响应房间需求、让汇报更可信

**技术基础**  
✅ `FacadeEnvelope`、`FacadeZone`、`orientationStrategy`已存在

**P1范围**
- [ ] 体量驱动立面分区
- [ ] 房间需求驱动开窗
- [ ] 朝向响应
- [ ] 窗墙比建议
- [ ] 简单遮阳/百叶策略
- [ ] 自动生成立面说明图与文字

**不要做太深**
- ❌ 复杂幕墙节点
- ❌ 构件级族库
- ❌ 施工级门窗排版

---

### P1-3 | 家具/设备适配反馈（Furniture / FF&E）

**价值**: 验证平面是否真能用，如果布不下去方案就不成立

**P1范围**
- [ ] 基础家具/设备模板库
- [ ] 自动布置建议
- [ ] 卫生间/办公/医疗/住宅典型布局
- [ ] 家具适配评分
- [ ] 基于布置结果反向提醒平面问题

**为什么是P1**: 先要有稳定的Plan与Compare，家具适配才有发挥空间

---

### P1-4 | 结构与设备的概念可行性示意

**价值**: 关键词是"可行性示意"，不是工程交付

**P1范围**

**Structure**
- [ ] 柱网建议
- [ ] 大跨风险提示
- [ ] 核心筒对位
- [ ] 标准层结构一致性提示

**Systems**
- [ ] 竖井位置
- [ ] 设备房服务逻辑
- [ ] 湿区靠井合理性
- [ ] 小透视/轴测示意
- [ ] 系统说明摘要

**不要做太深**
- ❌ 完整管综
- ❌ 施工级配筋/梁板
- ❌ 正式顾问交底模型

---

### P1-5 | Typology Packs

**价值**: 支持多业态，当前偏医疗不适合长期产品形态

**P1范围**
至少先支持:
- [ ] Healthcare
- [ ] Office
- [ ] Residential
- [ ] School

并联动:
- [ ] Room taxonomy
- [ ] Scoring config
- [ ] Program templates
- [ ] Presentation language
- [ ] Furniture library

**为什么是P1**: 先把"平台怎么工作"做对，比"支持多少业态"更重要

---

## 📦 P2: 保留能力，但现在不抢资源

这些不是没价值，而是不应成为当前主开发线

### P2-1 | 门窗表、设备表、细Schedule
- 现阶段只保留: 房间面积表、方案对比表、汇报图文清单
- 不投入产品化资源

### P2-2 | 深入IFC / BIM导出
- 保留基础导出架构
- 先把PDF/PPT/图片表达做好
- IFC留在概念交换级

### P2-3 | 施工级MEP / 深化结构
- 会把产品拖向另一条赛道
- 只做"可信示意 + 风险提示 + 设计表达辅助"

### P2-4 | 规范审图级别规则引擎
- 当前应是"设计辅助"，不是"正式审图"
- 过早推进会偏离方案阶段核心价值

---

## 📅 12个月排序节奏

### 第1阶段 (月1-4): 主闭环通
**目标**: 让用户能从资料进入→生成方案→比较方案→输出汇报

**优先做**
1. Import / Trace
2. Plan generation with constraints
3. Compare
4. Copilot proposal workflow
5. Presentation / PDF / PPT

---

### 第2阶段 (月5-8): 方案更可信
**目标**: 让方案不仅能讲，还更可信

**优先做**
1. Massing to floors
2. Facade concept
3. Furniture/FF&E feasibility
4. Structure / MEP conceptual diagrams
5. Typology packs

---

### 第3阶段 (月9-12): 产品边界稳
**目标**: 补足必要但不抢主线的支撑能力

**优先做**
1. 轻量schedules
2. 更好的导出
3. 更多模板
4. Review / Approval / Collaboration
5. 性能和稳定性优化

---
